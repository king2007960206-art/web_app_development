# PRD - 個人記帳簿系統

## 1. 專案概述

- **背景與動機**
  目前的記帳 App 往往充滿廣告或過於臃腫，且大多缺乏專業的財務邏輯。本專案旨在打造一個專為個人設計的記帳系統，強調操作簡便、介面清爽無廣告，並隱含專業的「借貸法則」會計邏輯，讓用戶能精確掌握資產淨值。
  
- **目標用戶**
  - 主要用戶：自己（個人記帳需求）。
  - 次要用戶：未來有共同分攤費用的室友、伴侶或小組成員。

- **核心價值主張**
  - 極速體驗：首頁醒目按鍵，3 秒內完成記帳。
  - 專業財務分析：以會計邏輯為核心，自動計算資產負債與淨值。
  - 介面與隱私：清爽簡約風設計，無廣告干擾，且提供嚴密的隱私保護機制。

## 2. 功能需求

以下列出系統的主要功能需求（User Stories）：

1. **快速記帳**：作為使用者，我希望能在首頁點擊醒目按鍵後，3 秒內完成「金額、類別、備註」的輸入，以便不耽誤日常時間。
2. **多帳戶管理**：作為使用者，我希望能分別建立與管理現金、銀行存款、行動支付（如 LINE Pay、街口）及信用卡帳戶，以便清楚掌握各平台的資金流動。
3. **收支分類**：作為使用者，我希望系統預設食、衣、住、行、育、樂等類別，並支持自定義標籤，以便有條理地紀錄每筆花費。
4. **預算設定與警告**：作為使用者，我希望能設定每月總預算或各類別預算，並在支出達 80% 時收到系統警告，以便控制開銷。
5. **資產負債表**：作為使用者，我希望系統能基於隱含的「借貸法則」會計邏輯，自動彙整所有帳戶，算出目前的資產淨值（資產 - 負債），以便了解真實財務狀況。
6. **視覺化圖表**：作為使用者，我希望能查看支出佔比的圓餅圖與收支趨勢的折線圖，並支持按週、月、年切換，以便直觀分析財務數據。
7. **週期性支出提醒**：作為使用者，我希望系統能自動記錄並提醒每月的房租、電信費或訂閱服務，以便避免遲繳或遺忘。
8. **搜尋與過濾**：作為使用者，我希望能根據關鍵字或日期區間搜尋與過濾歷史帳單，以便快速查閱特定紀錄。
9. **儲蓄目標設定**：作為使用者，我希望能設定儲蓄目標，並透過進度條顯示目前達成率，以便增加存錢的動力。
10. **小組/情侶共同記帳**：作為使用者，我希望能設定共同帳本，以便未來與室友或伴侶共同分攤房租或生活費。
11. **掃描發票/收據**：作為使用者，我希望能透過 OCR（文字識別）技術，拍攝收銀條或掃描電子發票 QR Code 自動帶入金額，以便減少手動輸入負擔。
12. **報表匯出**：作為使用者，我希望能將報表匯出為 CSV 或 Excel 檔案，以便在電腦上進行更複雜的二次開發與公式運算。
13. **外幣匯率換算**：作為使用者，我希望系統支援外幣匯率換算，以便準確記錄出國或海外消費。
14. **雲端同步與備份**：作為使用者，我希望能跨裝置登入並確保雲端同步，以便更換手機時資料不會遺失。
15. **隱私保護**：作為使用者，我希望系統包含密碼鎖或生物識別（FaceID/指紋）解鎖功能，以便保護個人財務隱私。

## 3. 非功能需求

- **技術限制**
  - 後端與架構：使用 Flask 作為網站框架。
  - 模板引擎：使用 Jinja2 渲染前端頁面。
  - 資料庫：使用 SQLite 作為關聯式資料庫。
- **效能與安全考量**
  - UI/UX：介面需走「清爽簡約風」，嚴禁任何干擾性廣告。
  - 資料安全：使用者的敏感資料與密碼須加密儲存，並藉由密碼鎖或生物辨識機制防止裝置端未授權存取。

## 4. MVP 範圍 (Minimum Viable Product)

根據系統複雜度，將需求劃分為以下階段：

### Must Have (第一階段：核心必備功能)
- 快速記帳（3秒內完成金額、類別、備註）
- 收支分類（預設與自定義標籤）
- 多帳戶管理（現金、存款、信用卡、行動支付）
- 資產負債表（隱含借貸法則，自動計算資產淨值）
- 清爽簡約的 UI 介面設計

### Should Have (第二階段：重要輔助功能)
- 視覺化圖表（圓餅圖、折線圖）
- 預算設定與 80% 超支警告
- 週期性支出設定與提醒
- 搜尋與過濾歷史帳單
- 基礎隱私保護（密碼鎖）

### Nice to Have (第三階段：進階加值功能)
- 雲端同步與備份（多裝置支援）
- 報表匯出（CSV/Excel）
- 掃描發票/收據（OCR 辨識）
- 外幣匯率換算
- 儲蓄目標進度條
- 小組/情侶共同記帳
- 進階隱私保護（FaceID/指紋生物識別）
